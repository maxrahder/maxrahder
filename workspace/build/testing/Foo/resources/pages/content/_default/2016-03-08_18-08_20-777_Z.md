Users themselves typically do *not* do testing &mdash; that responsibility is 
usually given to *test engineers*. 


<img src="resources/images/senchatest/RelationshipUserTestEngineer.jpg">


A test engineer understands user requirements
and can translate them into end-to-end tests, typically using the Event Recorder feature
built into Sencha Studio. Such tests may need tweaking and, therefore, a test
engineer may need some understanding of Jasmine and application code.
